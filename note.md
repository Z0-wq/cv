### 变量与基本数据类型（已掌握 2026-07-05）
**例题与参考答案：**
1. `x=10` 后又 `x="10"`，x最终类型？→ `str`（后赋值覆盖前面，动态类型）
2. 定义`score=95.5`，打印它的类型 → `score = 95.5` `print(type(score))`
3. `name = 小明`（不加引号）会报错吗？→ 会，`NameError`，因为不加引号时Python把"小明"当变量名去找，但从未定义过这个变量

**关键点：**
- 变量无需声明类型，直接赋值；同一变量可重新赋值为不同类型（动态类型），后赋值覆盖前面
- 基本类型：int整数、float浮点数、str字符串（必须加引号）、bool布尔值(True/False)
- `print()`打印内容，`type()`查看变量类型
- 语法规则：数字字符序列被Python直接解析为数字字面量；文字字母序列若不加引号，永远被当成变量名去查找（找不到则NameError），无法通过"提前声明"让不加引号的文字变成字符串内容——变量名和它存的值是两个独立的东西

**易错总结：**
- 一开始把Python的`print`和C语言的`printf`搞混，写成`printf(score)`——Python没有printf
- 第3题一开始只记住"要加引号"的规则，没说清楚原因；后自己推导出正确原因：不加引号=Python当成变量名去找，找不到才报错，且"变量名"和"变量存的值"是两个独立的东西，无法靠提前声明让裸文字变成字符串

### 运算符（已掌握 2026-07-05）
**例题与参考答案：**
1. `17 % 5` 和 `17 // 5` 分别是多少？→ `2` 和 `3`
2. `age == 18` 是赋值还是判断相等？→ 判断相等
3. `%`在图像处理里可能用来干什么？→ 每隔N个像素取一个点：`if i % N == 0:` 筛选出能被N整除的下标

**关键点：**
- 算术：`+ - * /`(除法结果永远是float)、`//`(整除，只取整数部分)、`%`(取余)
- 比较：`> < == !=`，结果永远是布尔值True/False；`=`是赋值，`==`才是判断相等，是新手最容易混淆的点
- `%`的核心应用——"每隔N个取一个点"：判断条件写`i % N == 0`，本质是筛选出所有能被N整除的数；留下来的这些数彼此间距恰好是N（因为都是N的倍数），间距由**除数N**决定，跟某次取余算出的具体余数值无关
- print()才是"显示"动作；单纯的表达式（如`age == 18`）在.py脚本里执行完不会自动显示结果，必须包在print()里才能看到；Jupyter Notebook的代码块最后一行例外，会自动显示

**易错总结：**
- 一开始把"某次取余算出的具体数值"误认为是"间隔值本身"（比如误以为`10%3=1`代表"间隔1"），后纠正为：间隔由除数N决定，`%`只是筛选"能否被N整除"的工具
- 主动提出`age==18`单独一行在.py脚本里不会显示结果，必须包一层`print()`才能看到——这个观察是对的，属于加分项而非错误

### 条件语句 if/elif/else（已掌握 2026-07-05）
**例题与参考答案：**
1. `age=15`时会打印什么？→ "青少年"（第一个if不满足，elif age>=12满足）
2. score>=60判断"通过/不通过" →
```python
if score >= 60:
    print("通过")

```
3. 判断num是否偶数 →
```python
if num % 2 == 0:
    print("偶数")
else:
    print("奇数")
```

**关键点：**
- 语法：`if 条件:` / `elif 条件:` / `else:`，冒号不能少，下一行必须缩进（缩进错会报错）
- 从上到下只执行第一个满足条件的分支，后面即使也满足也不会再执行
- **实用坑**：中文输入法下打的冒号是全角"："，Python只认英文半角":"，两者长得像但是不同字符，会导致SyntaxError——写代码时输入法要切到英文状态

**易错总结：**
- 写`else`时漏了冒号
- 用中文输入法打出全角冒号"："而不是英文半角":"，两者长得像但Python不认全角——已确认这是新手最容易踩的实际坑

### 循环 for/while（已掌握 2026-07-05）
**例题与参考答案：**
1. `for i in range(3, 15, 4):` 输出哪些数字？→ `3, 7, 11`（15不包含）
2. 从10开始每次减2，打印到0（包含0）→
```python
count = 10
while count >= 0:
    print(count)
    count -= 2
```
3. 指出死循环问题 → 缺少改变判断条件用的变量（如没写`count -= 1`），条件永远成立

**关键点：**
- `for i in range(start, stop, step):` 遍历一个范围，stop不包含在内；只写`range(n)`默认起点0步长1
- `while 条件:` 不知道循环几次、只知道满足什么条件就继续时用；必须在循环体里改变判断用到的变量，否则死循环
- 循环停止条件优先用范围判断(`>=`/`<=`)而不是精确相等(`==`/`!=`)，除非能保证一定精确命中该值，否则容易因步长/起点变化导致死循环或漏掉边界值
- "遍历"=把序列里每个元素一个不漏按顺序都过一遍

**易错总结：**
- 边界条件错误：写成`while count != 0`，导致循环在count变成0**之前**就退出，0从未被打印（差一错误）；最初用`!=-2`这种"精确凑数值"的方式绕过，虽然结果碰巧对，但不通用（步长/起点一变就可能失效）；最终理解应统一用`>= 0`这种范围判断更稳妥

### 函数定义与调用（已掌握 2026-07-05）
**例题与参考答案：**
1. 写函数`square(x)`返回x的平方 →
```python
def square(x):
    result = x * x
    return result
```
2.
```python
def greet():
    print("hello")

print("start")
```
只会打印什么？为什么？→ 只打印"start"，因为`greet()`从未被调用，函数体内的`print("hello")`不会执行
3. 函数没写return，调用后存入变量，这个变量的值是什么？→ `None`

**关键点：**
- `def 函数名(参数):` 定义函数，`return` 返回结果；没有return默认返回`None`（`None`是一个实实在在的值，打印出来会显示"None"这几个字母，不是啥都不显示）
- **核心区别，最容易搞混**：**"定义"（def写完）**和**"调用"（函数名()去触发执行）是两件完全独立的事**。函数定义完整与否、有没有return、写得好不好，都不影响它"有没有被执行"——唯一决定"是否执行"的，是有没有在代码里写`函数名()`去调用它。哪怕函数定义得再完整、再有意义，没被调用就是不会跑。

**易错总结：**
- 第2题一开始把"没有输出（没写return）"和"没有被调用"这两个完全独立的概念搞混，误以为"函数不完整/没有意义"是不执行的原因；后用"菜谱写完没人做"类比理清：定义（写菜谱）本身就是完整有意义的，执行与否只取决于有没有人调用（照着做）
- 第3题一开始说"什么也没有"，需注意`None`是一个具体的值，打印会显示"None"字样，不是空白无输出

### lambda与import（已掌握 2026-07-05）
**例题与参考答案：**
1. 用lambda判断一个数是否大于10 →
```python
panduan = lambda i: i > 10
num = 30
print(panduan(num))   # True
```
2. `import cv2` 和 `from cv2 import xxx` 的区别？→ 前者导入整个cv2库，后者只导入cv2库里指定的那个东西
3. 为什么喜欢`import numpy as np`起简称？→ 后续代码里写`np`更简洁方便

**关键点：**
- `lambda 参数: 表达式` 写临时/简短的函数，效果和def一样，适合只用一次的简单逻辑
- `import 库名` 导入整个库；`import 库名 as 简称`起简称（如`import numpy as np`是行业惯例写法）；`from 库名 import xxx` 只导入库里的某一个东西
- 视觉代码开头常见三件套：`import cv2` / `import numpy as np` / `import matplotlib.pyplot as plt`
- 函数/lambda的参数名只在其内部有效，外部访问不到——外部变量和参数即使撞名也互不影响，但调用时必须传外部实际存在的变量

**易错总结：**
- 第一次把变量/函数名取成`while`——`while`是Python保留关键字（用于while循环），不能拿来当变量名/函数名，会报语法错误；关键字如`if/for/def/return/while`等都不能作变量名
- "大于10"一开始写成`>=`（大于等于），与"大于"（严格不含等于）不符，需用`>`
- 调用时一度写成`panduan(i)`，但`i`只是lambda内部参数名，外部并不存在这个变量，应传外部实际定义的变量（如`num`）

### 类与对象（已掌握 2026-07-05）
**例题与参考答案：**
1. 根据Dog类造对象`dog3=Dog("大黄",2)`并调用bark() →
```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f"{self.name}在叫！")

dog3 = Dog("大黄", 2)
dog3.bark()
```
2. `dog1.age`和`dog2.age`是同一个东西吗？→ 不是，两个对象的属性完全独立
3. `self.name = name`左右两个name是同一个东西吗？→ 不是；右边是临时局部变量（调用时传入的值），左边是永久挂在对象身上的属性；本质是把临时输入永久存到对象身上

**关键点：**
- `class 类名:` 定义类（模板），根据模板`类名(参数)`造出的具体实例叫对象，每个对象互相独立
- `__init__(self, 参数...)` 是构造方法，造对象时自动执行，`self`代表"这个对象自己"，方法第一个参数永远写`self`（调用时不用手动传）
- `self.属性 = 参数`：右边参数是**临时的局部变量**（只在`__init__`执行时存在，装的是调用时传入的值），左边`self.属性`是**永久挂在对象身上的存储**，把临时传入的值永久存到对象自己身上——两者字面同名但不是同一个东西
- 不同对象的同名属性（如`dog1.age`和`dog2.age`）完全独立，互不影响
- `def`定义方法/函数，同样要以冒号结尾

**易错总结：**
- `def __init__(self, name, age)` 少写了结尾冒号——`def`同样要以冒号结尾
- 第一次回答"左右两个name的关系"时，只说清楚了左边是"对象自己的属性"，右边说得含糊（"代表这个属性是name"），追问后才精确表达出"临时输入 vs 永久存在"这个关键区分

### 继承（已掌握 2026-07-05）
**例题与参考答案：**
1. `dog.eat()`为什么能调用成功，Dog类里没写eat？→ 因为Dog继承了父类Animal的所有属性和方法
2. `class Cat(Animal): pass`，`Cat("咪咪").eat()`能跑吗？→ 能，因为`(Animal)`已完成继承，`pass`只是没有额外新增内容，不影响继承到的方法照常可用
3. 为什么PyTorch模型要写成`class MyModel(nn.Module):`继承形式？→ 避免重复实现一整套训练相关功能、直接复用官方更完善的现成实现，自己只需在此基础上添加模型具体结构，更简单省力

**关键点：**
- `class 子类(父类):` 括号里写父类名字即完成继承，子类自动拥有父类所有属性和方法，无需重写
- **继承只看`class 子类(父类):`这一行有没有写父类名字，跟子类身体里写不写内容（哪怕是`pass`空占位）完全无关**——`pass`只表示"没有额外新增内容"，不代表"没有继承"
- 子类可以新增自己独有的方法，也可以覆盖重写父类方法
- PyTorch写模型固定用`class MyModel(nn.Module):`——继承官方写好的`nn.Module`直接获得一整套现成的训练相关功能，避免重复造轮子且更完善，自己只需在此基础上写模型具体结构

**易错总结：**
- 第2题一开始把"pass（子类身体里没写新内容）"误判为"没有继承父类"，把"继承关系"和"子类身体里有没有新增代码"这两件独立的事搞混了；纠正后明确：继承只由`class 子类(父类):`这一行决定，与身体内容无关

### 路径操作 os（已掌握 2026-07-05）
**例题与参考答案：**
1. `os.listdir("images")`返回什么类型的数据？→ 一个列表，元素是文件名字符串
2. 遍历`"photos"`文件夹打印每个文件完整路径 →
```python
import os
folder = "photos"
files_path = os.listdir(folder)
for filename in files_path:
    full_path = os.path.join(folder, filename)
    print(full_path)
```
3. 为什么用`os.path.join`而不是字符串`+`拼接路径？→ 不同系统路径分隔符不同（Windows的`\`和Mac/Linux的`/`），`os.path.join`自动适配当前系统，写死斜杠换系统可能出问题

**关键点：**
- `os`是Python标准库模块，专门跟操作系统打交道（查文件、判断存在、拼路径等），需要`import os`才能用
- `os.listdir(文件夹路径)` 返回一个**列表**，列表里是该文件夹下所有**文件名字符串**（只是名字文字，不是文件内容本身）
- `os.path.join(文件夹, 文件名)` 拼接完整路径，比直接用字符串`+`拼接更安全——Windows用`\`分隔、Mac/Linux用`/`分隔，`os.path.join`自动适配当前系统，写死斜杠换系统可能出问题
- `os.path.exists(路径)` 判断文件/文件夹是否存在，返回True/False

**易错总结：**
- 第一次把返回值类型答成"图片"，混淆了"文件名文字"和"图片内容本身"——`os.listdir`只返回文件名字符串列表，不涉及图片实际内容
- 混用`pathlib.listdir(...)`——`listdir`是`os`模块的函数，`pathlib`模块没有这个函数，两个模块写法不能混用（当时只教了`os`的写法，`pathlib`只是提了名字没细讲）
- 变量命名不一致：存的变量叫`files_path`，循环时却写成从未定义过的`files`
- 路径字符串打字拼写错误（"photots"多打一个t），导致检查了一个不存在的路径——提醒要养成检查路径拼写细节的习惯

### 异常处理 try/except（已掌握 2026-07-05）
**例题与参考答案：**
1. 批量读100张图片，第30张损坏，不加/加了try/except分别会怎样？→ 不加：程序在第30张直接崩溃终止，后续处理和排查都受影响；加了：能捕获这次错误、打印具体原因、跳过继续处理剩下的图片
2. `except Exception as e:` 里的`e`是什么？→ 这一次具体发生的错误对象，存着报错的具体详情信息（不是错误类型本身）
3. `finally`和`else`的区别？→ `finally`不管有没有出错都执行（常用来收尾）；`else`只有在没有任何异常发生时才执行

**关键点：**
- `try:` 放可能出错的代码；`except 错误类型:` 指定发生某种特定错误时怎么处理，可写多个针对不同错误；`except Exception as e:` 万能兜底
- `e` 是变量名（exception的简写，习惯写法），它存的**不是错误类型，是这一次具体发生的错误对象/详情信息**——`FileNotFoundError`是错误的类型（种类），`e`是这一次具体报错说了什么（细节）
- `else`（可选）：try块**没有报错**时才执行；`finally`（可选）：**不管有没有报错都会执行**，常用来做收尾
- 批量处理文件时不加try/except，中途一个文件出错会导致整个程序崩溃终止；加了则能捕获错误、跳过继续处理，还能知道具体哪个文件、什么原因

**易错总结：**
- 第一次把`e`的作用理解成"代表错误类型"，需纠正为：`e`是变量名（exception简写），装的是**这一次具体的错误详情**，"类型"和"具体这次的详情"是两个不同层次（类型在`except`后面写，如`FileNotFoundError`；`e`是实例化后的具体报错信息）

### 线性代数——矩阵乘法（已掌握 2026-07-07）
**例题与参考答案：**
1. `A=[[1,0],[0,1]]`（单位矩阵），`B=[[3,4],[5,6]]`，`A@B`等于多少？→ `[[3,4],[5,6]]`（单位矩阵乘任何矩阵结果不变）
2. `A=[[2,1],[0,3]]`，`B=[[1,2],[3,4]]`，`A@B`第0行第1列的值？→ `8`（完整结果`[[5,8],[9,12]]`）
3. 矩阵乘法里"点乘"是什么意思？→ 第一个矩阵的第i行和第二个矩阵的第j列，对应位置相乘后全部加起来

**关键点：**
- 矩阵乘法（`@`）不是对应位置直接相乘（那是逐元素乘`*`），而是"行×列，逐项相乘再求和"
- 结果矩阵第i行第j列 = 第一个矩阵第i行 与 第二个矩阵第j列 做点乘
- 这个"点乘再求和"的运算，后面卷积、神经网络全连接层本质都是这个操作

### 线性代数——特征值/特征向量/SVD（已掌握 2026-07-07）
**例题与参考答案：**
1. 什么样的向量才叫"特征向量"？→ 被矩阵作用后**方向不变**（不变或正好反向），**只有长度会变**（被拉伸或压缩）的向量
2. 特征值等于3，意味着特征向量被拉伸/缩小了多少倍？→ 3倍
3. 为什么"抓大放小"（保留大特征值/奇异值，丢弃小的）能用来做图像压缩？→ 数值小的方向本身信息量很少，人眼几乎看不出差别，丢弃后数据量能大幅减少

**关键点：**
- 特征向量：矩阵作用后方向不变（或正好反向），只有长度按特征值倍数缩放的特殊向量；公式`A@v = λ*v`
- SVD（奇异值分解）：把特征值思想推广到任意形状矩阵（不要求方阵），奇异值越大代表该方向信息/变化越重要，越小代表越可忽略
- 图像压缩/PCA降维核心思路：抓大放小——只保留数值大的方向，丢弃数值很小的方向，信息损失很小但数据量大幅减少

**易错总结：**
- 第一次描述特征向量时前后表述矛盾（先说"改变方向不改变长度"，后又说"方向不变"），是把"方向"和"长度"谁变谁不变说反了；纠正为：方向不变、长度变

### 概率论——条件概率/贝叶斯/高斯分布（已掌握 2026-07-07）
**例题与参考答案：**
1. `P(生病|发烧)`表示什么意思？→ 在发烧的情况下，生病的概率
2. 贝叶斯公式的作用？→ 已知`P(A|B)`的情况下，反过来求`P(B|A)`
3. 高斯分布的图形是什么形状，哪个位置概率最高？→ 钟形，中间（平均值附近）概率最高，越往两边概率越低

**关键点：**
- 条件概率`P(A|B)`：在B已发生的前提下，A发生的概率——已知信息能缩小范围、改变概率估计
- 贝叶斯公式`P(B|A) = P(A|B)×P(B)/P(A)`：把已知方向的条件概率"反过来"算出你真正想要的方向；CV分类器中常用于"已知某特征出现的统计规律，反推看到该特征时属于某类别的概率"
- 高斯分布（正态分布）：钟形曲线，均值附近概率最高，越远概率越低；深度学习里权重初始化、噪声假设等常默认用高斯分布描述

### 微积分——导数/偏导数/链式法则/梯度（已掌握 2026-07-07）
**例题与参考答案：**
1. `y=x³`，导数`dy/dx`是多少？→ `3x²`
2. `z=3x²+y`，对x求偏导`∂z/∂x`是多少？→ `6x`（y当常数，求导后消失）
3. 为什么"反向传播"要用到链式法则？→ 反向传播从最后的loss开始，一层一层往前"倒着乘回去"算出每层参数怎么调整，这正是链式法则的连续应用

**关键点：**
- 导数：某一点上y随x变化的快慢程度，几何意义是该点切线斜率
- 偏导数：多变量函数只关心某一个变量变化时的导数，其余变量当常数处理
- 链式法则：`x→u→y`套娃式依赖关系下，`dy/dx = dy/du × du/dx`；神经网络是多层"套娃"，反向传播就是从loss开始逐层倒着应用链式法则，算出每层参数的调整方向
- 梯度：多变量情况下所有偏导数组成的向量，指向函数增长最快的方向；梯度下降是沿梯度**反方向**走，使loss减小

### 图像基础——像素/通道/色彩空间（已掌握 2026-07-07）
**例题与参考答案：**
1. 1080×720彩色图片，NumPy数组shape是多少？→ `(720, 1080, 3)`（高在前，宽在后，跟日常"宽×高"说法顺序相反）
2. 为什么找"红色区域"用HSV比RGB方便？→ RGB下红色在不同亮度组合千变万化难判断；HSV只需看H值是否在红色对应范围内，跟亮暗/鲜艳程度无关

**关键点：**
- 像素是图片最小组成单位，灰度图每像素1个数字（二维数组`(高,宽)`），彩色图每像素3个数字RGB（三维数组`(高,宽,3)`）

- HSV色彩空间：H色调/S饱和度/V明度；找特定颜色区域时比RGB更方便，因为只需判断H值范围，与亮暗无关
- **OpenCV读图默认是BGR顺序**（不是RGB），直接用Matplotlib等按RGB显示的工具会导致红蓝颠倒，需先转换顺序

**易错总结：**
- 第一次不清楚"1080×720"转成shape怎么表示，核心是没意识到shape的"高宽顺序"和日常说法"宽×高"是反过来的

### 图像滤波（已掌握 2026-07-07）
**例题与参考答案：**
1. 图片有很多"椒盐噪声"，优先选哪种滤波？为什么？→ 中值滤波，因为极端噪点值排序后会被挤到两端，取中间值时天然被排除
2. 均值滤波和高斯滤波的核心区别？→ 均值直接算周围像素平均值（边缘会被模糊掉，整体变糊）；高斯离中心像素越近权重越大，效果更自然
3. 人像美颜"磨皮但保留五官清晰"用哪种滤波思路？→ 双边滤波

**关键点：**
- 滤波核心思路：对每个像素看周围一小片区域（"核"），用周围信息重新计算该像素值，把突变的噪点拉回正常范围
- 均值滤波：周围区域直接算平均值，简单快速但边缘也会被模糊
- 高斯滤波：离中心越近权重越大，效果比均值更自然，是最常用的通用去噪方式
- 中值滤波：取排序后的中间值，对椒盐噪声（黑白点状噪点）特别有效
- 双边滤波：速度慢但能去噪同时保留边缘清晰（磨皮但五官清晰的思路）
- `(5,5)`等参数是"核大小"，核越大模糊效果越强、计算越慢

**名词解释：**
- `kernel`（核）：一个小矩阵（如5×5），像"小窗口"在图片上滑动，定义"每次处理时参考周围多大范围、按什么规则算"

### 边缘检测与形态学操作（已掌握 2026-07-07）
**例题与参考答案：**
1. Sobel和Canny哪个更常用、效果更精细？→ Canny
2. 二值图有很多细小白色噪点，用腐蚀还是膨胀去掉？→ 腐蚀
3. 想填补物体小黑色空洞又不想整体变形太多，用开运算还是闭运算？→ 闭运算

**关键点：**
- 边缘检测找的是"相邻像素亮度差异很大"的位置；一般先转灰度图再检测
- Sobel：计算亮度变化梯度找边缘，可分别只找横向/纵向边缘，相对简单
- Canny：多步骤组合算法（降噪→算梯度→筛选），效果比Sobel更干净精细，业内最常用
- 形态学操作用在二值图上：腐蚀让白色区域变瘦（去小白噪点），膨胀让白色区域变胖（填小黑空洞）
- 开运算=先腐蚀再膨胀，去小噪点且整体大小基本不变；闭运算=先膨胀再腐蚀，填小空洞且整体大小基本不变

**名词解释：**
- `cv2.imread("路径")`：读取图片文件，变成NumPy数组
- `cv2.CV_64F`：数据类型常量，表示结果用64位浮点数存——因为边缘检测算出的梯度值可能是负数，普通整数格式存不了负数，需要浮点数保住正负号和精度
- `cv2.Sobel(gray, cv2.CV_64F, 1, 0)`中的`1,0`：分别是"对x方向/y方向求几阶导数"，`1,0`=只对x方向求导（检测竖直边缘），`0,1`=只检测水平边缘
- `cv2.Canny(gray, 100, 200)`中的`100,200`：两个阈值，高于200肯定算边缘，低于100肯定不算，中间的看是否连着已确定边缘；数值调大更严格，调小更宽松
- `kernel`（核）：一个小矩阵，像"小窗口"在图片上滑动，定义处理时参考周围多大范围、按什么规则算

### 几何变换——仿射变换/透视变换（已掌握 2026-07-07）
**例题与参考答案：**
1. 仿射变换和透视变换，哪个能处理"平行线不再平行"的情况？→ 透视变换（仿射变换保证平行线依然平行，做不到这件事）
2. 确定仿射变换/透视变换各需要几个点？→ 仿射变换3个点，透视变换4个点
3. 歪的矩形名片矫正回正矩形，用仿射变换还是透视变换？→ 透视变换（角度导致的畸变，平行线本身就不平行，仿射变换无法处理）

**关键点：**
- 仿射变换：平移/旋转/缩放/错切的组合，**变换前后平行线依然平行**，只需3个点确定
- 透视变换：更自由的变换，**平行线变换后可能不再平行**（模拟斜着看的视觉效果），需4个点确定
- 拍照角度导致的"近大远小"透视畸变校正，必须用透视变换，仿射变换做不到（因为它保证平行不变，无法把歪四边形拉回正矩形）

**名词解释：**
- `cv2.getRotationMatrix2D((cols/2,rows/2), 45, 1)`：生成旋转变换矩阵，参数依次是旋转中心点、旋转角度、缩放比例（1=不缩放）
- `cv2.warpAffine(img, M, (cols,rows))`：把仿射变换矩阵M应用到图片img上，`(cols,rows)`是输出图片宽高
- `cv2.getPerspectiveTransform(pts1, pts2)`：告诉它"原图这4个点(pts1)变换后挪到新图这4个位置(pts2)"，算出透视变换矩阵
- `cv2.warpPerspective(img, M, (300,300))`：把透视变换矩阵应用到图片上，输出指定尺寸的结果

**易错总结：**
- 第一次把Q1的结论和理由对应反了：理由本身正确（仿射变换保证平行线依然平行），但据此应得出"仿射变换处理不了'不再平行'的情况"，答案应是透视变换，而不是仿射变换本身

### 特征检测与匹配（已掌握 2026-07-07）
**例题与参考答案：**
1. 为什么"角点"比"直线上的点"更适合做特征匹配？→ 角点独一无二、容易辨认，稍微移动就能看出明显不同，直线上的点左右移动看起来都差不多难以区分
2. ORB提取特征后返回的`kp`和`des`分别代表什么？→ `kp`(keypoints)是关键点位置，`des`(descriptors)是每个点对应的特征描述
3. `BFMatcher`里的"BF"是什么意思？→ Brute Force（暴力搜索），把每个特征描述去另一张图里逐一比对找最相似的
4. **编程题**：读取一张图，用ORB提取关键点，画出并显示 →
```python

import matplotlib.pyplot as plt

orb = cv2.ORB_create()
kp1, des1 = orb.detectAndCompute(img, None)

img_with_kp = cv2.drawKeypoints(img, kp1, None)
img_rgb = cv2.cvtColor(img_with_kp, cv2.COLOR_BGR2RGB)
plt.imshow(img_rgb)
plt.show()
```

**关键点：**
- 角点检测（Harris）：找图片里"独特、好辨认"的点——往任何方向移动窗口，画面都明显变化的点才是角点；平坦区域往哪移都不变，边缘只有一个方向移动会变
- ORB特征：不仅找到关键点位置，还给每个点生成一串"特征描述"（代表该点周围图案特征），方便与其他图对比
- 特征匹配（BFMatcher）：拿特征描述去另一张图里找最相似的点，配对起来，即"同一位置在两张图里分别对应的点"；**匹配需要两张图的描述互相比对，只有一张图时不涉及匹配**
- **原理串联**：Harris用Sobel算子（3.3学过）算每个像素x/y方向的梯度，再把梯度信息汇总成一个小矩阵，算这个矩阵的**特征值**（2.2学过）——两个特征值都大=角点，一个大一个小=边缘，两个都小=平坦区域

**名词解释：**
- `cv2.cornerHarris(gray, 2, 3, 0.04)`：`2`是blockSize（检测窗口大小，多大范围内判断是不是角点）；`3`是ksize（Sobel算子窗口大小，用来算梯度）；`0.04`是经验系数，一般固定用这个值
- `cv2.ORB_create()`：创建一个ORB特征提取器
- `orb.detectAndCompute(img, None)`：对图片做特征检测和描述计算，返回`kp`(关键点位置)和`des`(特征描述)
- `cv2.BFMatcher()`：创建一个"暴力搜索"匹配器；`.match(des1, des2)`把两组特征描述互相比对配对（需要两组描述，只有一组无法调用）
- `cv2.drawKeypoints(img, kp1, None)`：把关键点画在图上，**有返回值**（画好的新图），必须用变量接住，不能只调用不接收；参数位是"原图/关键点列表/输出图(None=自动生成)"，写代码时要填自己实际的变量名，不是照抄参数占位名

### 综合项目——图像矫正（已掌握 2026-07-08）
**编程题与参考答案：** 读取歪的文档照片，自动找角点，透视矫正成正矩形，并排显示对比 →
```python
import cv2
import numpy as np
import matplotlib.pyplot as plt

img = cv2.imread("document.jpg")
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
blur = cv2.GaussianBlur(gray, (5, 5), 0)

peri = cv2.arcLength(largest, True)
approx = cv2.approxPolyDP(largest, 0.02 * peri, True)

pts1 = np.float32(approx.reshape(4, 2))
pts2 = np.float32([[0,0], [500,0], [500,700], [0,700]])

plt.subplot(1, 2, 1)
plt.imshow(img_rgb)
plt.title("原图")

plt.subplot(1, 2, 2)
plt.imshow(warped_rgb)
plt.title("矫正后")

plt.show()
```

### 评估指标（已掌握 2026-07-09）

## 类别5：深度学习基础

### 神经网络基础（已掌握 2026-07-09）
**例题与参考答案：**
1. 不加激活函数只堆叠"矩阵乘法+加法"会怎样？→ 不管叠多少层，本质等价于一层线性运算（线性叠加还是线性），学不会复杂弯曲的规律
2. ReLU对`-3`和`5`分别输出什么？→ `0`和`5`
3. **编程题**：用NumPy实现sigmoid函数 →
```python
import numpy as np

def sigmoid(x):
    return 1 / (1 + np.exp(-x))

x = np.array([-1, 0, 1])
print(sigmoid(x))
```

**关键点：**
- 感知机公式：输出=输入×权重相加再加偏置，本质就是2.1学过的矩阵乘法（点乘）；全连接层就是`输出向量 = 权重矩阵 @ 输入向量 + 偏置向量`
- 激活函数必须存在：否则叠再多层网络都等价于一层线性运算，无法学习复杂规律
- ReLU：`f(x)=max(0,x)`，负数清零正数不变，简单快速最常用
- Sigmoid：`f(x)=1/(1+e^(-x))`，把任何输入压缩到0~1之间，常用于输出概率的场合

**名词解释：**
- `np.maximum(0, x)`：逐元素比较0和x，取较大的那个（用来实现ReLU）
- `np.exp(x)`：计算`e^x`（e是自然常数，约2.718）

**易错总结：**
- sigmoid公式漏写了关键部分：只写了`np.exp(x)`（即`e^x`本身），漏了"1加上它再用1除"的完整结构，也漏了指数上的负号（应为`e^(-x)`不是`e^x`），公式差一个负号结果完全不同

### 反向传播与优化器（已掌握 2026-07-09）
**例题与参考答案：**
1. 正向传播和反向传播分别在做什么，方向区别？→ 正向：输入一层层往前算出预测结果和loss；反向：从loss倒着往回走，用链式法则算每层参数该怎么调整；方向正好相反（前→后 vs 后→前）
2. 为什么反向传播必须用链式法则，不能跳过中间层直接算最前面参数的梯度？→ w1对loss的影响是间接的（w1→u→y→loss层层传递），两者间没有直接数学关系，微积分规则规定嵌套函数求导必须把每层导数依次乘起来，这是唯一数学上正确的方式，跳过中间层根本得不出正确结果（不是不准确，是算法错误）
3. SGD学习率设太大会怎样？→ 每一步走太大，容易走过头（在最优点附近来回震荡甚至无法收敛）

**关键点：**
- 正向传播：输入→逐层计算→预测结果→对比真实答案算出loss
- 反向传播：从loss开始，用链式法则（2.4学过）逐层倒着算出每层参数的调整方向和幅度；"反向"指计算方向与正向相反
- 链式法则应用：`x→u→y`套娃结构下，`∂loss/∂w1 = ∂loss/∂y × ∂y/∂u × ∂u/∂w1`，从loss往前连续乘
- 优化器执行"沿梯度反方向走一步"：SGD最朴素，`新参数=旧参数-学习率×梯度`；Adam更智能，自动调整每个参数各自的步子大小，收敛更快更稳，是目前最常用默认选择
- 学习率控制每步走多大：太大容易走过头（震荡/不收敛），太小学得太慢

**易错总结：**
- 第一次把"为什么用链式法则"的原因说成"结果更准确"，需纠正为：这是数学规则唯一正确的方式（w1对loss的影响是间接层层传递的，跳过中间层不是"不准"而是"算法本身错误"，没有其他可选方案

### 卷积核心操作——卷积/步长/padding（已掌握 2026-07-09）
**例题与参考答案：**
1. 5×5输入，3×3核，stride=1，无padding，输出尺寸？→ `(5-3+0)/1+1=3`，即3×3
2. 承接上题加padding=1，输出尺寸？→ `(5-3+2)/1+1=5`，即5×5
3. 传统OpenCV滤波核(手动指定)和CNN卷积核(学出来的)本质是同一种运算吗？→ 是，两者本质都是矩阵的点乘（对应位置相乘再求和），区别只在于核里的数字是人手动定的还是模型自己学出来的

**关键点：**
- 卷积运算=核在图片上滑动，每个位置"对应位置相乘再求和"（矩阵点乘，2.1学过），与3.2滤波用的"核"是同一种机制，区别是CNN的核数字由模型自己学出来，专门提取某种特征（边缘/纹理等）
- stride（步长）：核每次滑动挪多远，越大输出越小
- padding：在图片边缘补一圈值（通常补0），让边缘像素也能被充分扫到，同时控制输出尺寸
- 输出尺寸公式：`输出尺寸=(输入尺寸-核尺寸+2×padding)/步长+1`——算出来的是**边长**，不是总像素数；正方形输入+正方形核时，同一个边长同时适用于高和宽

**易错总结：**
- 把公式算出的"边长"误当成"总像素数"直接报数字（如答成9、25），需纠正为：公式结果是边长，输出应表示为"边长×边长"（如3×3、5×5）

### 感受野与池化（已掌握 2026-07-09）
**例题与参考答案：**
1. 网络叠的层数越多，感受野变大还是变小？为什么？→ 变大，因为后面层的一个点间接汇总了前面更多层的信息，"看到"的原图范围越叠越大
2. 最大池化和平均池化的核心区别？→ 最大池化：小窗口内只保留最大值；平均池化：小窗口内取平均值
3. **编程题**：对`(1,1,4,4)`的张量做`kernel_size=2,stride=2`最大池化，打印输出形状 →
```python
import torch
import torch.nn as nn

pool = nn.MaxPool2d(kernel_size=2, stride=2)
x = torch.randn(1, 1, 4, 4)
y = pool(x)
print(y.shape)   # torch.Size([1, 1, 2, 2])
```

**关键点：**
- 感受野：某个输出点"看到"了原图多大范围的信息；网络越深感受野越大，越靠后的层捕捉越全局/抽象的信息
- 池化作用：①减小计算量 ②增大感受野 ③保留主要特征、忽略细微位置变化（对物体小幅移动不敏感）
- 最大池化取窗口内最大值（保留最强信号），平均池化取平均值

**名词解释：**
- `nn.MaxPool2d(kernel_size=2, stride=2)`：创建一个最大池化层（工具/层对象本身），核大小2、步长2
- `torch.randn(1,1,4,4)`：生成指定形状的随机张量（PyTorch里"数据"叫张量tensor），这里形状是(批次数,通道数,高,宽)

**易错总结：**
- 反复出现"调用函数/层但不用变量接住结果"的模式（连续三次）：生成张量后不接住、池化层定义了但没真正调用处理数据、`print(torch.randn)`打印的是函数本身不是数据——核心要记住的模式是"下一步还要用的结果，必须写成`变量=函数(...)`存下来"

### 经典CNN架构——ResNet残差连接（已掌握 2026-07-09）
**例题与参考答案：**
1. 为什么网络叠太深，最前面的层反而学不到东西？→ 反向传播的链式法则要连续相乘传回最前面层，如果每次乘的数值都略小于1，层数一多梯度会几乎变成0（梯度消失），最前面的层收不到有效梯度
2. ResNet残差连接具体改了什么？→ 把没经过任何处理的原始输入，直接加到处理后的结果上：`输出 = F(输入) + 输入`
3. **编程题**：残差块，卷积层输入输出16通道、核5×5、padding=2 →
```python
import torch
import torch.nn as nn

class ResidualBlock2(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv = nn.Conv2d(16, 16, kernel_size=5, padding=2)

    def forward(self, x):
        out = self.conv(x)
        return out + x
```

**关键点：**
- 演进脉络：LeNet→AlexNet→VGG→ResNet，越往后越深、效果越好，但纯堆深度会遇到梯度消失问题
- 梯度消失：链式法则连续相乘，层数太深会导致最前面层收到的梯度趋近于0，学不到东西
- ResNet解决方案：残差连接`return out + x`，给梯度提供一条"抄近路"直接传回去，不完全依赖连续相乘那条路，缓解梯度消失，能训练上百层网络

**名词解释：**
- `super().__init__()`：调用父类(`nn.Module`)的构造方法，PyTorch要求这样写才能让内部机制（如自动追踪参数）正常工作
- `forward(self, x)`：定义数据正向传播时的处理逻辑，方法名固定为forward，是PyTorch规定
- `nn.Conv2d(输入通道,输出通道,kernel_size=,padding=)`：定义一个卷积层

**易错总结：**
- `def forward(self, x)` 漏写结尾冒号——`def`定义方法必须以冒号结尾，1.3学类时提过的规则在这里又出现一次

### PyTorch建模实战（已掌握 2026-07-10）
**例题与参考答案：**
1. `optimizer.zero_grad()`为什么每轮都要调用？→ PyTorch梯度默认会累加，不清零会导致梯度越来越大，每轮训练前必须先清零
2. `loss.backward()`和`optimizer.step()`分别对应哪两个概念？→ `backward()`对应反向传播（用链式法则算梯度）；`step()`对应优化器/梯度下降（拿到梯度后真正更新参数），两者是先后独立的两个步骤
3. **编程题**：`SimpleNet2`（20输入3输出），测试正向传播 →
```python
import torch
import torch.nn as nn

class SimpleNet2(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc = nn.Linear(20, 3)
    def forward(self, x):
        return self.fc(x)

model = SimpleNet2()
x = torch.randn(5, 20)
outputs = model(x)
print(outputs.shape)   # torch.Size([5, 3])
```

**关键点：**
- `nn.Linear(输入特征数,输出特征数)`：全连接层（5.1学的权重矩阵@输入+偏置），PyTorch自动创建权重和偏置参数
- `nn.CrossEntropyLoss()`：分类常用损失函数；`torch.optim.Adam(model.parameters(), lr=...)`：优化器，`model.parameters()`是模型所有可学习参数
- 训练循环固定顺序：`optimizer.zero_grad()`(清梯度) → `outputs=model(x)`(正向传播) → `loss=criterion(...)`(算loss) → `loss.backward()`(反向传播算梯度) → `optimizer.step()`(更新参数)
- 保存/加载：`torch.save(model.state_dict(), 路径)`保存参数；加载时先造同结构空模型，再`model.load_state_dict(torch.load(路径))`填入参数

**名词解释：**
- `epoch`：把全部训练数据完整过一遍叫一个epoch
- `model.parameters()`：模型里所有需要学习/更新的参数集合

**易错总结：**
- 实例化时用错类名（定义了`SimpleNet2`却创建`SimpleNet()`的实例）
- 把训练循环相关内容（labels、epoch循环、optimizer.zero_grad()）混进只需要测试正向传播的任务里，其中`optimizer`还没定义会直接报错
- `print(model.shape)`——把"模型本身"和"正向传播的输出结果"搞混，模型没有`.shape`，只有具体的张量数据才有，应打印`outputs.shape`

### 训练技巧（已掌握 2026-07-10）
**例题与参考答案：**
1. Dropout为什么能对抗过拟合？→ 随机屏蔽一部分神经元，逼着模型不能依赖任何单一神经元，必须学会更普遍、更鲁棒的规律
2. BatchNorm解决的核心问题？→ 每层输出数值范围忽大忽小导致训练不稳定难收敛；BatchNorm把数据拉回稳定范围（均值0方差1附近），让训练更快更稳
3. **编程题**：`__init__`定义BatchNorm(8通道)和Dropout(0.3)，`forward`里依次经过conv→bn→dropout →
```python
class SimpleNet(nn.Module):
    def __init__(self):
        super().__init__()
        # self.conv 假设已存在
        self.bn = nn.BatchNorm2d(8)
        self.dropout = nn.Dropout(0.3)

    def forward(self, x):
        out = self.conv(x)
        out = self.bn(out)
        out = self.dropout(out)
        return out
```

**关键点：**
- 数据增强：对同一张图做随机翻转/旋转/裁剪/调亮度等，生成"变体"训练样本，减少过拟合
- BatchNorm：把每层输出数据拉回稳定范围（均值0方差1附近），训练更快更稳
- Dropout：训练时随机让一部分神经元输出变0，逼模型学更普遍的规律，专门对抗过拟合
- 学习率调整策略：训练前期学习率大（快速接近答案），后期学习率变小（微调避免震荡）
- **重要规则**：所有带参数的层（卷积/BatchNorm/Dropout等）必须在`__init__`里定义一次，`forward`只负责调用；若定义在`forward`里，每次正向传播都会重新创建全新随机初始化的层，无法正常训练，也无法被正确识别为模型的一部分

**名词解释：**
- `nn.BatchNorm2d(通道数)`：批归一化层，参数是对应输入的通道数
- `nn.Dropout(丢弃率)`：丢弃率如0.3表示训练时随机让30%的神经元输出变0
- `torch.optim.lr_scheduler.StepLR(optimizer, step_size=, gamma=)`：学习率调度器，每隔step_size个epoch把学习率乘以gamma

**易错总结：**
- 把BatchNorm和Dropout层定义在了`forward`方法里而不是`__init__`里——违反PyTorch核心规则，会导致每次正向传播都重新随机初始化这些层，无法正确训练
- 定义了bn和dropout却没有真正调用它们处理数据，结尾又误用了上一节残差连接的`return out+x`模式，与本题要求的"conv→bn→dropout→返回"流程不符

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class SimpleCNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(3, 16, kernel_size=3, padding=1)
        self.pool = nn.MaxPool2d(2, 2)
        self.conv2 = nn.Conv2d(16, 32, kernel_size=3, padding=1)
        self.fc1 = nn.Linear(32 * 8 * 8, 10)

    def forward(self, x):
        x = self.pool(F.relu(self.conv1(x)))
        x = self.pool(F.relu(self.conv2(x)))
        x = x.view(x.size(0), -1)
        x = self.fc1(x)
        return x

model = SimpleCNN()
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)

for epoch in range(2):

        optimizer.zero_grad()
        outputs = model(images)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()
```

**整体逻辑串讲（先懂大局，再看细节）：**

整个事情要干什么：我们想做一个程序，让它"看一张图，猜出这是10个类别里的哪一个"。要做到这件事，只需要两步：
1. **先造一个"猜测机器"**（也就是模型/`SimpleCNN`这个类）——这一步只是设计蓝图，告诉它"数据进来之后该怎么一步步处理"，但光有蓝图什么都不会发生
2. **拿真实数据去"训练"这个机器**——让它看很多张"图+正确答案"，每看一次就微调一下自己内部的参数，猜得越来越准

**为什么训练部分要放在最后**：必须先把"猜测机器"造出来（`class SimpleCNN`定义完，并且`model = SimpleCNN()`真的造出一个实例），损失函数和优化器也要提前准备好（这些是"训练时要用的工具"），这些准备工作必须在前面，"真正开始训练"这个动作自然只能放在最后——跟"`def`定义完不等于执行，必须调用才执行"是同一个道理：前面全是"准备"，训练循环才是"真正开始干活"。

**epoch是什么，为什么要循环两层**：想象有一整套100张图片的"教材"。把这100张图全部看一遍，叫1个epoch。为什么要看好几遍（这里设的是2遍）：看一遍很可能学得不够好，就像人复习课本，看一遍记不住，多看几遍印象才深——训练模型也是这个道理，反复用同样的数据练习，模型才会越来越准。但100张图不能一次性全塞给模型算（电脑内存扛不住，效果也不好），所以要"一批一批"喂，比如每批32张——这就是`trainloader`做的事：把100张图拆成好几个"32张一批"的小份。所以两层循环的意思是：
- 外层`for epoch in range(2)`：把整套教材（所有图片）从头看到尾，重复2遍
- 内层`for images, labels in trainloader`：每遍看的时候，是一批一批（每批32张）看完的，`images`是这一批的图片，`labels`是这一批图片对应的正确答案

**训练循环里那5行，用"做练习题"来比喻**：
1. `optimizer.zero_grad()`：擦掉上一批练习题剩下的草稿纸痕迹——不擦的话，这次算的"错在哪"会跟上一批的混在一起，越滚越乱（PyTorch的梯度默认是累加的，必须先清空）
2. `outputs = model(images)`：让"猜测机器"看这一批图片，说出它的猜测答案（正向传播）
3. `loss = criterion(outputs, labels)`：拿机器猜的答案，跟真正正确的答案对比，算出"这次错得有多离谱"这个分数（loss，4.1学的概念）
4. `loss.backward()`：根据"错得多离谱"，反推出机器内部每一个参数该往哪个方向调整（反向传播，链式法则，5.2学的）
5. `optimizer.step()`：真正按照第4步算出的方向，把每个参数微调一点点（优化器真正动手改参数，5.2学的梯度下降）

这5行是一个完整的"练习一批题→改正一次"的动作，内层循环让这个动作对着每一批数据都做一遍，外层循环让整个过程重复2轮（相当于把整套题再练一遍）。

**`lr=0.001`是什么**：`lr` = learning rate（学习率，5.2学过）。控制第5步"调整参数"时，每次调整的步子有多大。0.001是个比较小、比较保守的步子——每次只微调一点点，不会因为一步迈太大而"矫枉过正"。这个数字是常见的经验默认值，不是必须精确等于这个。

---

**架构设计逻辑：为什么卷积→池化→卷积→池化→全连接**

**为什么卷积和池化要交替堆叠**：第一层卷积核比较小（比如3×3），只能"看到"图片里很小一块区域，所以第一层提取的都是很局部、很基础的特征——比如某个方向的边缘、简单的颜色/纹理变化。池化把图片"压缩"变小（比如32×32变成16×16）。这样做完之后，下一层卷积虽然还是用同样大小的核（3×3），但因为图片本身变小了，这个3×3核相对能"看到"原图更大的一片区域——这就是感受野（5.4学的）在起作用：每叠一层（尤其配合池化），感受野越来越大。所以"第二层卷积"做的事，是在"第一层已经提取出的基础特征"之上，组合出更复杂、更抽象的模式——比如第一层看到的是"边缘"，第二层能看到"由几条边缘组成的形状/纹理"（比如猫耳朵的轮廓）。

**类比人认猫的过程**：先看到一些零散的线条、色块（第一层卷积，局部、基础）→再把这些线条组合，看出"这是尖尖的耳朵形状""这是一片毛茸茸的纹理"（第二层卷积，范围更大、更抽象）→层数越深、池化次数越多，"看到的范围"越大，提取的特征越接近"整个物体的形状/类别"这种高层概念。所以卷积负责"由简单到复杂，一层一层挖掘更抽象的特征"，池化负责"压缩数据、同时帮下一层卷积间接扩大能看到的范围"——两者交替，是为了逐步从"像素级细节"走到"物体级概念"。

**为什么最后要接全连接层**：前面的卷积+池化，做的事始终是局部性的——不管叠多少层，它关注的还是"图片某个区域长什么样"。但最终我们要的是一个全局判断：这整张图，到底是10个类别里的哪一个。全连接层的作用，就是把前面提取到的所有特征（不管是图片哪个区域提取出来的），全部汇总起来，综合考虑，给出一个最终的、针对整张图的判断——这也是为什么要先用`x.view()`把多维的特征图"拉平"成一条向量，因为全连接层要做的正是"不分区域，把所有特征放在一起统一权衡"。

**一句话总结**：卷积+池化 = 局部特征提取器（从细节到抽象，逐步放大视野）；全连接层 = 全局决策器（把所有提取到的特征汇总，做出最终分类判断）。这就是为什么几乎所有图像分类CNN都是"卷积池化堆几层，最后接全连接"这个固定套路。

---

**最终理解确认：**

整体就是建立一个机器，然后训练它的过程。在建立机器的过程中，先提取特征，再将其整合汇总。具体对应：
- **`forward`的目的**：它就是"猜测机器"真正干活的说明书——定义了"一张图片进来，具体要经过哪些步骤（卷积→激活→池化→...→全连接），最后变成10个类别的分数"这个完整的加工流程。`__init__`里定义的`conv1`、`pool`、`fc1`只是"准备好的零件"，`forward`才是"把这些零件按顺序组装起来，真正让数据流过去"的地方。
- **训练循环部分**：相当于开始用公式函数来训练——更精确地说，这一部分是真正让机器"变聪明"的过程：不是简单套公式算一次就完了，而是反复（每一批数据、每一个epoch）执行"猜一次→看错多少→算出往哪调→真正调一点点"，一轮一轮下来，模型内部的参数被持续微调，猜测就越来越准。

**整体逻辑总结**：造机器（模型结构+forward）→ 准备训练工具（loss+optimizer）→ 反复喂数据让它变聪明（训练循环）。

**易错总结：**
- 本节综合项目难度陡增（涉及模型架构设计逻辑+完整训练流程），第一次讲解只做了逐行翻译，未讲清楚"整体在干什么、为什么这样排布代码顺序"，导致跟不上；后改为先讲整体目标和架构设计逻辑（为什么卷积池化交替、为什么最后接全连接、训练循环的"练习题"类比），再对应到具体代码行，理解才跟上——**这是一个通用的学习方法提示：遇到复杂综合代码，先搞懂整体在解决什么问题、大致分几步，再逐行看细节，而不是反过来**

## 类别6：目标检测与YOLO系列

### 任务分类概念（已掌握 2026-07-11）
**例题与参考答案：**
1. 同一张街道照片，分类/检测/分割输出分别是什么？→ 分类："这是街道"（一个标签）；检测："汽车在(x1,y1,x2,y2)、行人在(x3,y3,x4,y4)"（框+标签，可能多个）；分割：精确到每个像素属于街道/汽车/行人/背景中的哪一类
2. 为什么检测输出"不固定长度"，分类是固定的？→ 一张图可能有0个物体也可能有15个物体，模型要自己判断"到底有几个框"；分类不关心位置和数量，只给一个整体标签
3. 只需知道"有没有汽车"，不需要位置和数量，用哪种任务？→ 图像分类，因为不关心具体位置，也不关心有没有别的物体

**关键点：**
- 分类：输出一个标签，不关心位置和数量
- 检测：输出多个"框+标签"，既要说"有什么"又要说"在哪"，可能同时检测出多个物体
- 分割：输出精确到每个像素的类别归属，比检测框更精细
- 检测比分类难的核心原因：输出数量不固定（0个到多个物体都可能），模型需自己判断框的数量

### 检测核心概念——bbox/anchor/IoU/NMS/mAP（已掌握 2026-07-11）
**例题与参考答案：**
1. `(0,0,10,10)`和`(5,5,15,15)`两个框的交集面积？→ 重叠区域坐标`(5,5,10,10)`，宽=10-5=5，高=10-5=5，面积=5×5=25
2. NMS怎么去掉重复框？→ 保留置信度最高的框，把跟它IoU超过某阈值的其他框全部删掉，只留最可信的那一个
3. mAP和精确率/召回率的关系？→ 每个类别可算出一条精确率-召回率曲线，曲线下面积是该类别的AP；所有类别AP取平均就是mAP

**关键点：**
- bbox：描述框位置，常见`(x1,y1,x2,y2)`(左上右下坐标)或`(x,y,w,h)`(左上坐标+宽高)两种格式
- IoU=交集面积/并集面积，衡量两个框重叠程度，1=完全重合，0=完全不重叠；判断预测框准不准、判断两框是否框同一物体
- NMS：对同一物体的多个重叠预测框，只保留置信度最高的，删除跟它IoU超阈值的其余框
- anchor：提前预设的不同大小/长宽比候选框模板，模型基于这些模板做微调而不是从零猜框
- mAP：每类别精确率-召回率曲线下面积(AP)的平均值，检测模型最常用综合评分标准

**易错总结：**
- 算交集面积时只算出了重叠区域的坐标范围，忘记把坐标换算成"宽×高"得到最终面积数值

### 检测网络发展脉络——two-stage vs one-stage（已掌握 2026-07-11）
**例题与参考答案：**
1. two-stage为什么慢，核心原因？→ 第二阶段要对每一个候选区域（可能上千个）都单独过一遍网络判断类别，相当于同一个网络被重复调用成百上千次，这才是计算量暴增的核心原因（不只是"分两步"本身）
2. YOLO"You Only Look Once"体现的核心思想？→ 不分两步，直接一次性同时预测框的位置和类别，把检测问题变成回归问题
3. 摄像头实时检测该选one-stage还是two-stage？→ one-stage，因为整张图只需过一次网络，不用对成百上千候选区域重复计算，速度更快，适合实时场景

**关键点：**
- two-stage（如R-CNN系列）：先找候选区域（第一阶段），再对每个候选区域单独判断类别+精修框（第二阶段）；精度通常较高，但因第二阶段要重复调用网络成百上千次，速度慢
- one-stage（YOLO）：一次性同时预测框位置和类别，把检测变成回归问题；整张图只过一次网络，速度快，适合实时检测；现代YOLO精度已接近two-stage同时保持速度优势

**易错总结：**
- 第一次把"two-stage慢"的原因归结为"分两步筛查"本身，未指出真正核心：第二阶段要对每个候选区域（数量可能上千）都重复调用网络做判断，这才是计算量暴增、速度慢的根本原因

### YOLO环境搭建与官方Demo（已掌握 2026-07-11，含实操验证）
**例题与参考答案：**
1. `YOLO("yolo11n.pt")`里的"n"代表什么？→ nano，最小最快的版本（还有s/m/l/x等更大更准但更慢的版本）
2. 为什么用ultralytics库不需要自己写数据加载/训练循环？→ 库已经把这些封装好了，直接调用即可
3. **编程题**：加载模型对图片推理并打印检测框数量 →
```python
from ultralytics import YOLO

model = YOLO("yolo11n.pt")
results = model("street.jpg")
print(len(results[0].boxes.xyxy))
```

**关键点：**
- `YOLO("yolo11n.pt")`加载官方预训练模型（已认识COCO数据集80种常见物体：人/车/猫狗等），本地没有会自动下载
- `model(图片路径)`直接推理；`results`是列表（支持多图输入），`results[0]`取第一张图结果
- `results[0].show()`显示检测结果图；`.boxes.xyxy`所有框坐标(对应6.2的bbox格式)、`.boxes.conf`置信度、`.boxes.cls`类别编号
- 想知道检测框数量用`len(results[0].boxes.xyxy)`（1.5学的len），不是`len(results)`（那是"处理了几张图"）

**实操记录：**
- 已用VS Code+Jupyter Notebook实际跑通完整流程，成功检测出图片中的目标并画出框
- **实操中遇到的真实坑**：Windows路径含反斜杠`\`，直接写字符串会被Python当成转义字符解析导致SyntaxError（unicode error），解决方法是在路径字符串前加`r`前缀（如`r"C:\Users\..."`），告诉Python这是原始字符串不做转义处理
- **真实误检案例**：一张白色卷毛狗的照片被模型识别成"teddy bear"（置信度0.74）而非"dog"——因为毛茸茸白色卷毛的视觉特征与泰迪熊玩偶相近，是模型预训练数据里容易混淆的一对类别；这类误检分析会在6.7"结果分析"进一步系统学习

**易错总结：**
- 图片文件名沿用上一节示例("photo.jpg")未改成本次任务要求的文件名
- 变量名反复少写"s"：定义的是`results`(复数)，多次误写成`result`(单数，未定义过)
- `for`循环遍历打印每个框坐标 ≠ 题目要求的"打印框的数量"，应直接用`len()`；且`len(results)`算的是"图片数量"不是"框数量"，要查的是`results[0].boxes.xyxy`里的元素个数
- `for ... in ...`遗漏结尾冒号
