## 阶段一：编程与环境基础 🛠️
**时间段：07-04 ~ 07-10（第1周）**

**本阶段目标**：拥有可用的开发环境，能独立写出处理图片文件的Python脚本。

**为什么这一步最先做**：视觉学习后面所有代码都要靠Python+NumPy跑，这一阶段是纯地基，跳过或学得不牢，后面每一步都会卡壳。

**核心是什么**：不是"记住语法"，而是**能独立把一个想法写成能跑的代码**——比如"批量读取一个文件夹里的图片并做点简单处理"，这件事你要能不看教程独立做出来。

**知识点清单（按顺序学）：**

1. **环境搭建**：安装Anaconda/Miniconda，学会创建/切换虚拟环境，用pip/conda装包（比如装opencv-python）。
   资料：Anaconda官方文档 → docs.anaconda.com
2. **编辑器与Notebook**：VS Code装Python和Jupyter插件，搞清楚.py脚本和.ipynb笔记本的使用场景区别。
   资料：VS Code官方Python教程 → code.visualstudio.com/docs/python/python-tutorial
3. **Git/GitHub基础**：会用git提交代码、推送到GitHub仓库，以后每个项目都要托管在这里当作品集。
   资料：Pro Git中文版（全文免费） → git-scm.com/book/zh/v2
4. **Python基础语法**：变量类型、运算符、if/for/while、列表/字典/元组/集合。
   资料：廖雪峰Python教程 → liaoxuefeng.com/wiki/1016959663602400
5. **函数与模块化**：把重复代码封装成函数，学会import自己写的模块。
   资料：同上廖雪峰教程对应章节
6. **面向对象基础（类）**：为什么这个重要——后面PyTorch的每个模型都是"继承一个类"写出来的，这里不打好基础，后面直接看不懂模型代码。
   资料：廖雪峰教程"面向对象编程"章节
7. **文件读写与异常处理**：批量读写图片文件、处理路径、try/except捕获报错。
   资料：Python官方文档 → docs.python.org/zh-cn/3/tutorial
8. **NumPy矩阵运算**：为什么这个是核心——**图像在计算机里就是一个数字矩阵**，你对图像做的所有操作（调亮度、翻转、裁剪）本质上都是对NumPy数组做运算，这是贯穿整个视觉学习的底层语言。
   资料：NumPy官方Quickstart → numpy.org/doc/stable/user/quickstart.html
9. **Matplotlib可视化**：把图片和处理结果显示出来，后面看模型训练曲线也靠它。
   资料：Matplotlib官方教程 → matplotlib.org/stable/tutorials

**验收标准（本阶段结束前必须能独立做到）：**
- [ ] 独立创建一个conda环境并装好常用库
- [ ] 独立写一个脚本：批量读取文件夹图片 → 用NumPy调整亮度 → 用Matplotlib显示前后对比
- [ ] 独立把这个小项目推送到自己的GitHub仓库

## 阶段二：数学基础支撑 📐
**时间段：07-11 ~ 07-17（第2周，穿插在阶段一/三之间的碎片时间进行，不单独占整块时间）**

**本阶段目标**：不是重新系统学一遍数学教材，而是**扫清后面看深度学习公式、理解反向传播时会卡住的数学盲点**。

**核心是什么**：**链式法则和梯度下降**——这是"反向传播"这个深度学习最核心机制的全部数学本质。这一个点没吃透，后面学CNN、学PyTorch训练模型都会变成"死记硬背调库"，遇到问题不知道从哪改。

**知识点清单：**

1. **线性代数——矩阵与向量运算**：矩阵乘法、转置、逆矩阵。
   为什么重要：图像=矩阵，卷积运算本质上是矩阵运算的一种变体。
   资料：《Mathematics for Machine Learning》免费电子书第2章 → mml-book.github.io
2. **线性代数——特征值/特征向量/SVD**：
   为什么重要：图像降维、压缩、PCA背后都是这套理论。
   资料：同上电子书第4章
3. **概率论——条件概率/贝叶斯/高斯分布**：
   为什么重要：分类模型输出的"概率"、损失函数的设计假设，背后都是概率论。
   资料：《深度学习》(花书)中文免费版第3章 → exacity.github.io/deeplearningbook-chinese
4. **微积分——导数/偏导数/链式法则/梯度**：**本阶段最核心的一点**。
   为什么重要：神经网络"学习"的过程就是不断算梯度、沿梯度反方向调整参数，链式法则就是让这个计算在多层网络里传递下去的数学工具。
   资料：花书第4章 + mml-book第5章

**验收标准：**
- [ ] 手算一个3×3矩阵乘法，并用NumPy验证结果一致
- [ ] 能用自己的话解释"为什么SVD/特征值能用来降维"
- [ ] 能用贝叶斯公式手解一道简单的分类概率题
- [ ] 手推一个简单复合函数（比如 y = (2x+1)² ）的导数，体会链式法则

## 阶段三：传统计算机视觉（OpenCV） 👁️
**时间段：07-18 ~ 07-31（第3-4周）**

**本阶段目标**：独立完成一条"从读图到几何校正"的完整传统图像处理流水线，这是你第一次真正做"视觉"。

**核心是什么**：**理解"图像处理=对矩阵做数学操作"，以及相机成像的几何模型（标定/畸变校正）**。这一块对你有硬件背景的人尤其关键——以后接摄像头做硬件项目，测距离/测角度全靠这里的知识。

**知识点清单（按顺序学）：**

1. **图像基础**：像素、通道、RGB/HSV/灰度色彩空间转换。
   为什么重要：这是后面一切算法输入的数据表示方式。
   资料：OpenCV官方Python教程 → docs.opencv.org/4.x
2. **图像滤波去噪**：均值/高斯/中值/双边滤波的原理和区别。
   为什么重要：真实摄像头拍的图有噪声，滤波几乎是所有视觉pipeline的第一步预处理。
   资料：同上OpenCV官方教程对应章节
3. **边缘检测与形态学操作**：Sobel/Canny边缘检测、腐蚀膨胀/开闭运算。
   为什么重要：找轮廓、找标定板角点等传统检测算法都建立在这个基础上。
   资料：OpenCV官方教程 + PyImageSearch博客实战案例 → pyimagesearch.com
4. **几何变换**：仿射变换、透视变换。
   为什么重要：摄像头拍的图往往是歪的或带视角畸变的，要校正成"正视图"才能做后续测量分析。
   资料：OpenCV官方教程"Geometric Transformations"章节
5. **特征检测与匹配**：Harris角点、ORB特征、特征匹配。
   为什么重要：图像拼接、目标跟踪、视觉SLAM这些应用都基于"找到两张图里的同一个点"这个能力。
   资料：OpenCV官方教程"Feature Detection"章节
6. **相机标定与畸变校正**：内参/外参、棋盘格标定法。
   为什么重要（对你尤其重要）：真实镜头都有畸变，做测距/定位类硬件项目前必须先标定摄像头，否则测出来的数据全是错的。
   资料：OpenCV官方教程"Camera Calibration"章节
7. **🏆综合项目：图像矫正**：串联以上所有知识点，完成"歪的照片 → 找四个角点 → 透视变换矫正成正矩形"的完整流程。

**验收标准：**
- [ ] 独立完成RGB↔HSV↔灰度互转
- [ ] 给一张带噪声的图去噪，对比效果
- [ ] 独立提取一张图的完整边缘轮廓
- [ ] 独立标定一个摄像头并校正畸变
- [ ] 不看教程独立跑通"图像矫正"完整项目

## 阶段四：机器学习与深度学习原理 🧠
**时间段：08-01 ~ 08-14（第5-6周）**

**本阶段目标**：理解神经网络为什么"能学习"，能手推反向传播的计算过程，能用PyTorch训练一个基础CNN模型。

**核心是什么**：**卷积操作原理 + 反向传播机制**——这两点是深度学习一切应用（包括后面的YOLO）的数学地基。不吃透这两点,你后面用YOLO就只是在"盲调一个黑箱工具"，出问题不知道怎么改。

**知识点清单（按顺序学）：**

1. **机器学习基本概念**：训练/测试集划分、过拟合欠拟合、损失函数、梯度下降。
   资料：Google Machine Learning Crash Course文字版 → developers.google.com/machine-learning/crash-course
2. **评估指标**：准确率/精确率/召回率/F1/混淆矩阵——后面看检测模型效果全靠这套指标。
   资料：scikit-learn官方Metrics文档 → scikit-learn.org/stable
3. **神经网络基础**：感知机、全连接层、激活函数（ReLU/Sigmoid）。
   资料：《动手学深度学习》(D2L)文字版第3-4章 → zh-v2.d2l.ai
4. **反向传播与优化器**：链式法则如何在多层网络里逐层传递梯度、SGD/Adam优化器原理。
   资料：D2L第4-5章 + 花书第6章
5. **卷积神经网络原理**：**本阶段最核心的知识点**——卷积核如何在图像上滑动提取特征、步长/padding、感受野、池化操作。
   为什么最核心：这是CNN能"看懂"图像的根本原理，YOLO的骨干网络全部建立在这个操作上。
   资料：D2L第6章"卷积神经网络"
6. **经典CNN架构演进**：LeNet→AlexNet→VGG→ResNet，重点理解ResNet的残差连接解决了什么问题（梯度消失，让网络能做得更深）。
   建议直接读一遍原始论文体会作者的思路，不用逐字抠细节：
   论文：《Deep Residual Learning for Image Recognition》(ResNet) → arxiv.org/abs/1512.03385
7. **PyTorch框架入门与实战**：Tensor张量操作、autograd自动求导、搭建/训练/保存/加载模型的完整代码流程。
   资料：PyTorch官方教程 → pytorch.org/tutorials
8. **🏆综合项目：CIFAR-10图像分类**：从零搭建一个CNN，在CIFAR-10数据集上训练出一个能用的分类模型。

**验收标准：**
- [ ] 手推一个两层神经网络的前向传播
- [ ] 手推一次简单网络的反向传播梯度计算
- [ ] 手算一次卷积操作的输出尺寸
- [ ] 能说清楚ResNet残差结构解决的问题
- [ ] 不看教程独立用PyTorch搭建并训练出CIFAR-10分类模型

## 阶段五：目标检测与YOLO系列 🎯
**时间段：08-15 ~ 08-28（第7-8周）**

**本阶段目标**：能自己标注数据集、训练自定义YOLO检测模型、看懂并分析训练结果。

**核心是什么**：**IoU/NMS/mAP这套检测评价体系，以及YOLO"单阶段回归"的核心思想**——这是你从"只会做图像分类"跨越到"能做目标检测"的关键一步，也是你暑假规划里最贴近实际应用的部分。

**知识点清单（按顺序学）：**

1. **任务类型区分**：图像分类 vs 目标检测 vs 图像分割，搞清楚三者输出形式的本质差异（一个标签 vs 框+标签 vs 像素级掩码）。
   资料：Ultralytics官方文档"Tasks"页 → docs.ultralytics.com/tasks
2. **检测核心概念**：边界框(bbox)、锚框(anchor)、交并比(IoU)、非极大值抑制(NMS)、平均精度均值(mAP)。
   资料：Ultralytics官方"YOLO Performance Metrics"文档 → docs.ultralytics.com
3. **检测网络发展脉络**：two-stage（先找候选区域再分类，如R-CNN系列）vs one-stage（直接一步回归出框，如YOLO）。
   建议读一遍YOLO原始论文，体会作者"把检测问题当回归问题来解"的思路转变：
   论文：《You Only Look Once》(YOLO) → arxiv.org/abs/1506.02640
4. **YOLO环境搭建与官方Demo**：安装ultralytics库，跑通官方预训练模型（YOLOv8/YOLO11）做推理。
   资料：Ultralytics官方Quickstart → docs.ultralytics.com/quickstart
5. **数据标注**：给自己收集的图片标注检测框。
   工具/资料：Roboflow（在线标注平台） → docs.roboflow.com，或本地工具LabelImg
6. **自定义数据集训练与调参**：配置数据集yaml文件、调整epoch/batch size/学习率等超参数、监控训练过程。
   资料：Ultralytics官方"Train"文档 → docs.ultralytics.com/modes/train
7. **模型评估与结果分析**：看懂mAP曲线、混淆矩阵，分析误检/漏检的原因并给出改进方向。
   资料：Ultralytics官方"Val"文档

**验收标准：**
- [ ] 举例说明分类/检测/分割输出的区别
- [ ] 手算两个矩形框的IoU
- [ ] 独立跑通一次官方预训练模型的推理
- [ ] 独立标注至少20张图片并导出数据集
- [ ] 独立训练出一个自定义检测模型
- [ ] 独立分析一次训练结果并指出可改进的地方

## 阶段六：综合项目与部署 🚀
**时间段：08-29 ~ 09-04（第9周）**

**本阶段目标**：把前面五个阶段的所有能力串成一个完整、可展示的项目，结合你的硬件背景做出差异化。

**核心是什么**：**工程闭环能力**——从摄像头采集画面，到模型推理出结果，再到结果被下游使用（显示/传给单片机控制硬件），这一整条链路能不能自己独立跑通，是检验"这个暑假是否真的学会了"的唯一标准，比任何理论考核都更有说服力。

**知识点清单：**

1. **摄像头实时推理集成**：把YOLO模型接入摄像头视频流，做实时检测。
   资料：Ultralytics官方"Predict"文档 → docs.ultralytics.com/modes/predict
2. **嵌入式部署（可选，视硬件条件而定）**：把模型部署到树莓派/Jetson Nano/K230等边缘设备上，结合你的单片机基础做下位机联动（比如识别到目标后控制云台或小车动作）。
   资料：NVIDIA Jetson官方文档 → docs.nvidia.com/jetson （其他硬件按厂商SDK文档）
3. **项目文档化与GitHub整理**：写清楚README、整理requirements.txt、代码结构规范化，让项目"可复现、可展示"。
   资料：GitHub官方"About READMEs"文档 → docs.github.com ，Git操作复习 → git-scm.com/book/zh/v2

**验收标准：**
- [ ] 独立实现摄像头实时检测并显示结果
- [ ] （如硬件条件允许）完成边缘设备部署，实现视觉+硬件联动
- [ ] 项目发布到GitHub，README能让别人看懂怎么跑起来
- [ ] 能完整讲清楚这个项目从头到尾每一步的原理，而不是"代码能跑但不知道为什么"
